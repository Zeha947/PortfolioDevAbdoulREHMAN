TP8

Animation & Sprites Dynamiques

Analyse de l'Exercice 2 : Mario Run (CSS)

Connaissances mises en jeu : Cet exercice exploite les @keyframes pour gérer deux animations parallèles : le défilement infini du décor et la course du personnage. L'utilisation de steps(5) est ici fondamentale pour découper visuellement la planche de sprites de Mario et créer un mouvement fluide.

Difficulté : Moyen. La précision du timing entre le défilement du fond et l'animation des jambes est essentielle pour éviter un effet de glissade.

Intérêt pédagogique : Excellent pour comprendre les bases du "parallax scrolling" et de l'animation cyclique en pur CSS.

Qualité du résultat : Le rendu est dynamique et fidèle aux mécaniques des jeux de plateforme classiques.

Analyse de l'Exercice 3 : Animation de Sprite via JavaScript

Connaissances mises en jeu : Contrairement au CSS pur, cet exercice délègue le contrôle de l'animation à JavaScript en utilisant un objet de données externe infoSprite.js. Le script doit parcourir les 13 frames définies (de 0 à 12) pour mettre à jour en temps réel la position du fond et les dimensions de l'élément créé au clic.

Difficulté : Difficile. L'obligation d'utiliser les données de l'objet impose une rigueur dans la gestion des boucles d'animation et du nettoyage du DOM (remove) après l'effet.

Intérêt pédagogique : Utile pour apprendre à manipuler des structures de données complexes (JSON/Objets) afin de piloter des éléments graphiques.

Qualité du résultat : Malheureusement je n'ai pas réussis à faire marcher l'exercice dans le site de la page web du TP8
