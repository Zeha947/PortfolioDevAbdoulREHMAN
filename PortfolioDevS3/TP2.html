Analyse de l'Exercice 2.1 : Grid par numéros de lignes

Connaissances mises en jeu : Cet exercice utilise le module CSS Grid Layout en se basant sur la numérotation des lignes (`grid-column: 1 / 4`). Il a fallu définir précisément la grille parente avec `grid-template-columns` et `grid-template-rows` pour placer chaque item.

Difficulté : Facile. Le concept de lignes est intuitif une fois le schéma de la grille dessiné.

Intérêt pédagogique : C'est utile car comprendre comment un élément s'étend sur plusieurs colonnes ou lignes est fondamental pour sortir des layouts simples.

Qualité du résultat : Le code est explicite et le résultat visuel est parfaitement conforme à la structure demandée.

Analyse de l'Exercice 2.3 : Imbrication Flexbox

Connaissances mises en jeu : Reproduction d'un layout complexe uniquement avec `display: flex`. Cela demande une imbrication de conteneurs (`#middle`, `#center`, `#right`) pour gérer les directions horizontales et verticales.

Difficulté : Moyen. La logique de conteneurs parents/enfants est plus complexe à gérer ici qu'avec Grid pour un résultat identique.

Intérêt pédagogique : L'exercice m'a montré que Flexbox est parfait pour des alignements unidimensionnels, Grid est bien plus adapté aux structures bidimensionnelles.

Qualité du résultat : Malgré la complexité de l'arborescence HTML, le rendu est fluide et respecte les dimensions imposées.

Analyse de l'Exercice 3 : Layout complexe en 5x5

Connaissances mises en jeu : Création d'une grille 5x5 avec des unités en pourcentage (`20%`). Placement d'éléments avec des enjambements complexes sur les lignes et colonnes.

Difficulté : Moyen. Il fallait que je fasse attention sur les indices de début et de fin (`grid-column: 1 / 5` par exemple) pour éviter les chevauchements non désirés.

Intérêt pédagogique : Comprendre le grid et son utilisation

Qualité du résultat : Le rendu est propre et les couleurs permettent de bien identifier chaque zone de la structure.
